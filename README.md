## What This Vendor Onboarding Template Covers

| Onboarding Area | What The Buyer Needs |
| --- | --- |
| Vendor Onboarding Summary | Record buyer, procurement owner, vendor setup owner, package, and `$203` team-license amount |
| Supplier Intake Details | Capture vendor name, contact, billing contact, delivery contact, invoice path, and receipt path |
| Product And License Line Item | Describe the Agent Ops Command Center paid ZIP, seven seats, license scope, and delivery artifact |
| Billing And Compliance Notes | Track billing requirements, security review, data boundary, support contact, and internal vendor record |
| Approval Evidence | Tie manager approval, budget approval, security review, quote, PO, and invoice evidence to onboarding |
| Checkout Readiness | Separate vendor setup intent from checkout, receipt, payout, or seller-dashboard proof |
| Delivery And Acceptance | Preserve ZIP digest, delivery manifest, buyer acknowledgement, and acceptance owner |
| Revenue Proof | Prevent vendor setup, page views, stars, forks, or downloads from being counted as revenue |

## Revenue Rule

Revenue is not counted from this repository, page views, stars, forks, template use, order intent, vendor onboarding, supplier intake, or preview downloads.

Revenue is proven only by checkout, receipt, payout, or seller-dashboard evidence tied to the paid template pack.
